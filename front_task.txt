Создай фронтенд на Next.js 15 (App Router) + Tailwind CSS + TypeScript.
Сервис: "Morf" — еженедельные челленджи для художников по созданию персонажей.

=== DOCKER (ОБЯЗАТЕЛЬНО) ===

Вся работа с проектом — через Docker. На хосте нет Node.js/npm/pnpm.
Все команды (install, dev, build) — внутри контейнера.

Требования:
- Dockerfile для Next.js
- docker-compose.yml (или docker-compose.dev.yml)
- Контейнер в сети существующих контейнеров бэкенда
- Название сети: morf-network (внешняя, уже созданная)
- Порты: 3000:3000
- Volume: .:/app (hot reload в dev)

Пример docker-compose:
  services:
    frontend:
      build: .
      container_name: morf-frontend
      networks:
        - morf-network
      ports:
        - "3000:3000"
      volumes:
        - .:/app
        - /app/node_modules
      command: pnpm dev

  networks:
    morf-network:
      external: true

Команды для работы:
- docker compose up --build -d
- docker compose exec frontend pnpm install
- docker compose exec frontend pnpm dev
- docker compose exec frontend pnpm add [пакет]
- docker compose exec frontend sh (консоль внутри)

=== АДАПТИВ (КРИТИЧНО: художники работают на планшетах) ===

Три брейкпоинта, все три первоклассные:

Desktop (≥1280px):
- Лента /feed: 4 колонки masonry
- Главная: референсы в ряд, полноразмерные карточки
- Модалки: центрированные, max-width 800px
- Хедер: полная навигация

Tablet (768px — 1279px) — ПРИОРИТЕТ:
- Лента /feed: 3 колонки masonry (или 2 если портретная ориентация)
- Главная: референсы сеткой 2x2 или горизонтальный скролл
- Модалки: 90vw ширины, удобные для тача
- Хедер: компактный, иконки крупнее (min 44x44px для тача)
- Кнопки: min-height 48px, padding увеличен
- Dropzone: крупнее, с явной зоной для пальца
- Отступы: больше воздуха между элементами

Mobile (<768px):
- Лента /feed: 1-2 колонки
- Главная: вертикальный стек, карточки на всю ширину
- Модалки: fullscreen или 95vw
- Хедер: только лого + иконка профиля
- Нижняя навигация (опционально) вместо боковой

Общие правила для планшета:
- Все интерактивные элементы min 44x44px
- Hover-эффекты заменены на active-стейты (тач-устройства)
- Горизонтальный скролл только там, где он ожидается (галереи)
- Шрифты: не меньше 16px для читаемости
- Изображения: tap-to-zoom на планшете

=== СТИЛЬ: ГОТИКА / DARK FANTASY ===

Цветовая схема (тёмная, без переключения):
- Фон: #0a0a0f
- Поверхности: #14141f, #1e1e2e
- Акцент: #8b0000 (тёмно-красный)
- Текст: #e2e2e2 (основной), #8b8b9a (вторичный)
- Границы: 1px solid #2a2a3a
- Тени: glow акцентным цветом

Типографика:
- Заголовки: Cinzel (Google Fonts), uppercase, tracking-wider
- Текст: Inter
- Кнопки и навигация — заглавными

Элементы:
- Карточки: минимальное скругление (2px) или острые углы
- Кнопки: тонкая рамка, glow при active/hover
- Декор: SVG-силуэт готической архитектуры на фоне (opacity 0.15)

=== API ===
Базовый URL: /api
Авторизация: X-Device-Token (header)

=== ЭНДПОИНТЫ ===
- POST /auth/register — nickname → { token, recovery_code }
- POST /auth/recover — recovery_code (12 символов) → { token }
- GET /reference-sets — список сетов
- GET /reference-sets/{id} — сет с изображениями
- GET /artworks?reference_set_id= — лента работ (approved only)
- POST /artworks — multipart загрузка
- POST /artworks/{id}/likes — тоггл лайка

=== СТРАНИЦЫ ===

1. LAYOUT:
   - Хедер: "MORF" (Cinzel, uppercase, tracking-[0.3em]), справа — профиль/вход
   - Фон: #0a0a0f + декоративный SVG готической архитектуры

2. AUTH МОДАЛКА:
   - Вкладки: "Войти" / "Регистрация"
   - Регистрация: nickname → после успеха показать recovery_code:
     * Заголовок: "СОХРАНИТЕ КОД ВОССТАНОВЛЕНИЯ"
     * Код крупным monospace, кнопка "Копировать"
     * Чекбокс "Я сохранил код" → активирует "Продолжить"
   - Вход: recovery_code → "ВОССТАНОВИТЬ ДОСТУП"

3. ГЛАВНАЯ "/" — Текущий челлендж:
   - Селектор недели (дропдаун в готической рамке)
   - Карточки референсов: изображение + описание
   - Кнопки: "УЧАСТВОВАТЬ" (красная рамка, glow) и "СКАЧАТЬ РЕФЕРЕНСЫ"
   - "Участвовать" → модалка загрузки

4. МОДАЛКА ЗАГРУЗКИ:
   - Dropzone с пунктирной рамкой, glow при drag
   - Поле caption (опционально)
   - Кнопка "ОТПРАВИТЬ" → toast "Работа отправлена на модерацию"

5. ЛЕНТА "/feed":
   - Masonry сетка (4 колонки десктоп, 3 планшет, 1-2 мобильные)
   - Фильтр по неделе
   - Карточка: превью, автор, ♥ + счётчик
   - Клик → модалка: полноразмерное изображение, лайк, закрыть

=== ТЕХНИКА ===
- React Query + Zustand (auth: token + recovery_code в localStorage)
- shadcn/ui (переопредели стили под готику)
- Google Fonts: Cinzel, Inter
- Next/Image
- Tailwind breakpoints: sm:640 md:768 lg:1024 xl:1280

=== ПОРЯДОК ===
1. Создай Dockerfile и docker-compose.yml
2. Запусти контейнер: docker compose up --build -d
3. Внутри контейнера: shadcn init, установи зависимости
4. Настрой тему (globals.css с переменными готики)
5. Layout + шрифты + фон
6. Auth модалка с recovery code
7. Главная с референсами
8. Лента /feed

Начни с шага 1-3. Создай Dockerfile, docker-compose.yml, запусти контейнер и покажи структуру папок.
